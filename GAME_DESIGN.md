# 게임 기획서 — AI Mini Arcade

> Claude API를 활용해 매 플레이마다 새로운 콘텐츠가 생성되는 세로형 모바일 미니게임 모음

---

## 1. 게임 개요

| 항목 | 내용 |
|------|------|
| **타이틀** | AI Mini Arcade |
| **장르** | 미니게임 모음 (캐주얼) |
| **플랫폼** | iOS / Android (세로형 9:16) |
| **핵심 컨셉** | Claude API가 게임 콘텐츠를 실시간 생성 — 퀴즈 문제, NPC 대사, 이야기 전개가 매 판마다 달라짐 |
| **타겟** | 10~30대, 짧은 시간에 즐기는 AI 체험 게임 |

## 2. 미니게임 상세 기획

### 2-1. 퀴즈 게임 (AI Quiz)

**목표**: Claude가 출제한 문제를 맞혀 점수를 획득한다.

**플레이 흐름**
1. 플레이어가 주제(역사 / 과학 / 상식 / 문화 / 스포츠) 선택
2. Claude가 해당 주제의 4지선다 문제 5개 생성 (문제 + 보기 4개 + 정답 + 해설 포함)
3. 각 문제에 20초 제한시간 타이머 표시
4. 정답 선택 시 → Claude가 짧은 해설 텍스트 표시
5. 오답 선택 시 → 정답 강조 + 해설 표시
6. 5문제 종료 후 점수 집계 → Claude가 총평 코멘트 출력

**Claude 역할**
- 문제 생성: 주제에 맞는 난이도 중상 문제를 JSON 형식으로 반환
- 해설 제공: 정답/오답에 따라 적절한 설명 출력
- 총평 코멘트: **디시인사이드 커뮤니티 스타일** 말투로 출력 (욕설 제외)
  - 예시: "야 이게 틀리면 진짜 ㅋㅋ", "오 5문제 다 맞았네 실화냐", "3개 맞았는데 이 정도면 뭐 나쁘진 않지"

**승패 조건**
- 5문제 중 3개 이상 정답 → 클리어
- 2개 이하 → 실패 (재도전 유도)

---

### 2-2. NPC 대화 게임 (Talk to NPC)

**목표**: Claude NPC와 대화를 통해 힌트를 모아 미션을 완료한다.

**플레이 흐름**
1. 시나리오 설정 화면: 배경(판타지 마을 / 우주정거장 / 탐정 사무소 등) 선택
2. Claude가 NPC로서 첫 대사와 함께 미션 제시
   - 예: "용사님, 마을에 도둑이 들었어요. 어제 밤 뭔가 보셨나요?"
3. 플레이어가 텍스트 입력으로 NPC와 자유 대화
4. 대화 중 Claude가 미션 달성 여부를 내부적으로 판단
5. 조건 충족 시 → 미션 성공 메시지 출력
6. 10턴 내 미션 미달성 시 → 실패

**Claude 역할**
- NPC 페르소나 유지: 설정한 배경에 맞는 말투와 성격으로 일관되게 응답
- 힌트 제공: 플레이어가 막히면 자연스럽게 단서 언급
- 미션 판정: 대화 내용 기반으로 성공/실패를 판단해 JSON 응답에 포함

**승패 조건**
- 10턴 이내 미션 키워드 달성 → 클리어
- 10턴 초과 → 실패

---

### 2-3. 스토리 릴레이 (Story Relay)

**목표**: Claude와 번갈아 이야기를 이어가며 짧은 소설을 완성한다.

**플레이 흐름**
1. 장르 선택 (판타지 / 로맨스 / 공포 / SF / 일상)
2. Claude가 첫 문장으로 이야기 시작
   - 예: "비가 내리던 어느 밤, 오래된 카페 문이 삐걱거리며 열렸다."
3. 플레이어가 다음 문장(1~3문장) 입력
4. Claude가 이어받아 2~3문장 추가
5. 5라운드 반복 후 Claude가 마무리 문단 + 제목 생성
6. 완성된 이야기를 텍스트로 복사/공유 가능

**Claude 역할**
- 이야기 흐름 유지: 플레이어 입력의 분위기와 방향을 이어가되 적절히 전개
- 마무리 및 제목 생성: 전체 흐름을 감안한 엔딩과 제목 출력

**승패 조건**
- 없음 (완성 자체가 목표, 재미와 공유에 초점)

## 3. UI/UX 설계

### 화면 구성 — YouTube Shorts / Instagram Reels 방식

```
┌─────────────────┐
│                 │
│   미니게임 A    │  ← 풀스크린 카드 1
│   (퀴즈)        │
│                 │
│  [재시작] [공유]│
└─────────────────┘
       ↕ 스와이프
┌─────────────────┐
│                 │
│   미니게임 B    │  ← 풀스크린 카드 2
│   (NPC 대화)    │
│                 │
└─────────────────┘
```

- **위아래 스와이프**로 다음/이전 게임으로 전환 (페이지 스냅)
- 각 카드는 독립 실행 — 스와이프 전환 시 현재 게임 상태 초기화
- AI 응답 대기 중 → 점 세 개 로딩 애니메이션 표시 (타이핑 효과)
- 하단 고정 버튼: **재시작** / **공유** (공유 시 OS 네이티브 Share Sheet 호출)

### 해상도 기준
- 기준 해상도: 1080 × 1920 (9:16)
- 안전 영역(Safe Area) 반영 필수 (노치, 다이나믹 아일랜드 대응)

## 4. Claude API 활용 전략

| 게임 | API 호출 시점 | 시스템 프롬프트 핵심 |
|------|-------------|-------------------|
| 퀴즈 | 게임 시작 시 일괄 생성 (5문제) | "JSON 배열로 퀴즈 5개 반환. topic, question, choices[4], answerIndex, explanation 포함" |
| 퀴즈 총평 | 게임 종료 후 1회 | "디시인사이드 스타일, 욕설 없이, 점수에 맞는 반응" |
| NPC 대화 | 플레이어 입력마다 1회 | "NPC 페르소나 유지, 미션 달성 여부를 JSON isMissionComplete 필드에 포함" |
| 스토리 릴레이 | 플레이어 입력마다 1회 / 마무리 1회 | "전체 흐름 유지, 마지막 라운드엔 엔딩 + 제목 생성" |

**오프라인 폴백**
- API 연결 실패 시 각 게임별 로컬 예시 데이터로 대체 실행
- 사용자에게 "오프라인 모드로 실행됩니다" 안내 토스트 표시

## 6. 제약 사항 / 리스크

| 항목 | 내용 | 대응 |
|------|------|------|
| API 응답 지연 | 모바일 네트워크에서 1~3초 지연 가능 | 로딩 인디케이터 + 타임아웃 설정 (10초) |
| 오프라인 환경 | API 미연결 시 게임 불가 | 로컬 예시 데이터로 폴백 |
| API 비용 | 과도한 호출 시 비용 발생 | 응답 캐싱, 불필요한 재요청 방지 |
| 콘텐츠 부적절성 | Claude가 의도치 않은 응답 생성 | 시스템 프롬프트에 가이드라인 명시 |
| API Key 노출 | 클라이언트 빌드에 키 포함 위험 | 개발/테스트 전용, 출시 시 서버 중계 방식으로 전환 필요 |
